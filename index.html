This is a quick start with slog4s on the JVM with logback backend. It doesn’t mean we are limited to JVM or logback! Au contraire!

Installation

Add new library dependencies to your build.sbt

libraryDependencies ++= Seq("com.avast" %% "slog-api" % "0.0.0+8-8a1e226e-SNAPSHOT", 
                            "com.avast" %% "slog-slf4j" % "0.0.0+8-8a1e226e-SNAPSHOT")

We will be using logback with logstash encoder. So make sure it’s in your dependency list and it’s configured properly.

<configuration>
    <appender name="STDOUT" class="ch.qos.logback.core.ConsoleAppender">
        <encoder class="net.logstash.logback.encoder.LogstashEncoder" />
    </appender>
    <root level="INFO">
        <appender-ref ref="STDOUT" />
    </root>
</configuration>

Usage

Obligatory imports before we start:

import cats.effect._
import cats.syntax.all._
import slog._
import slog.slf4j._

Let’s start by creating a LoggerFactory that is backed by slf4j (and logback under the hood). With that, we can create a named Logger.

val loggerFactory = Slf4jFactory[IO].noContext.make
// loggerFactory: LoggerFactory[IO] = slog.slf4j.Slf4jFactory$Slf4jFactoryBuilderWithContext$$anon$1@7cc6eb29
val logger = loggerFactory.make("test-logger")
// logger: Logger[IO] = slog.Logger@4a2e0bca

It’s finally time to log something!

logger.info("Hello world!").unsafeRunSync()

Output:

{
  "@timestamp" : "2020-01-03T12:08:14.243Z",
  "@version" : "1",
  "level" : "INFO",
  "level_value" : 20000,
  "logger_name" : "test-logger",
  "message" : "Hello world!",
  "thread_name" : "run-main-0",
  "x-file" : "index.md",
  "x-line" : 41
}

That was pretty boring, except x-file and x-line attributes that denote location of the log message within a file. We can also provide an exception:

logger.error(new Exception("Boom!"), "Something went horribly wrong.").unsafeRunSync()

Let’s make our message more content rich with additional arguments:

logger.info
      .withArg("string_value", "<VALUE>")
      .withArg("bool_value", true)
      .withArg("list_value", List(1,2,3))
      .log("Message with arguments")
      .unsafeRunSync()

Output:

{
  "@timestamp" : "2020-01-03T12:08:14.295Z",
  "@version" : "1",
  "bool_value" : true,
  "level" : "INFO",
  "level_value" : 20000,
  "list_value" : [
    1,
    2,
    3
  ],
  "logger_name" : "test-logger",
  "message" : "Message with arguments",
  "string_value" : "<VALUE>",
  "thread_name" : "run-main-0",
  "x-file" : "index.md",
  "x-line" : 53
}

Additional arguments are type safe. Following code will file in compile time:

class That(value: String)
logger.info
      .withArg("that_value", new That("value"))
      .log("Does not compile")
      .unsafeRunSync()
// error: could not find implicit value for evidence parameter of type slog.StructureEncoder[repl.Session.App.That]
// logger.info
// ^

However there is built-in support for case classes and sealed traits provided by generic module.
