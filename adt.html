ADT support

slog4s provides built-in support for automatic derivation of StructureEncoder typeclass, which allows one to use case class or sealed trait as additional arguments for logging. It supports both fully automatic derivation, and semi automatic derivation.

Installation

libraryDependencies ++= Seq("com.avast" %% "slog-generic" % "0.0.0+8-8a1e226e-SNAPSHOT")

Example

Suppose we have following case class:

case class Foo(fooValue: String)
case class Bar(barValue: Int, foo: Foo)

val bar = Bar(42, Foo("Hello!"))

Automatic derivation

With automatic derivation you just need to include proper import.

import slog.generic.auto._
logger.info
      .withArg("bar", bar)
      .log("Logging bar instance")
      .unsafeRunSync()

Output:

{
  "@timestamp" : "2020-01-03T12:08:16.932Z",
  "@version" : "1",
  "bar" : {
    "barValue" : 42,
    "foo" : {
      "fooValue" : "Hello!"
    }
  },
  "level" : "INFO",
  "level_value" : 20000,
  "logger_name" : "test-logger",
  "message" : "Logging bar instance",
  "thread_name" : "run-main-0",
  "x-file" : "adt.md",
  "x-line" : 41
}

Semi automatic derivation

Sometimes it might be more convenient to have StructureEncoder instance have defined directly in a code. You can use semi automatic derivation for that:

import slog.generic.semi._

object Foo {
  implicit val fooEncoder: StructureEncoder[Foo] = structureEncoder[Foo]
}

object Bar {
  implicit val barEncoder: StructureEncoder[Bar] = structureEncoder[Bar]
}

logger.info
      .withArg("bar", bar)
      .log("Logging bar instance")
      .unsafeRunSync()

Output:

{
  "@timestamp" : "2020-01-03T12:08:16.944Z",
  "@version" : "1",
  "bar" : {
    "barValue" : 42,
    "foo" : {
      "fooValue" : "Hello!"
    }
  },
  "level" : "INFO",

  "logger_name" : "test-logger",
